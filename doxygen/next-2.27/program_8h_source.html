00003 /*
00004  * A compiled program consists of several data blocks, all allocated
00005  * contiguously in memory to enhance the working set. During compilation,
00006  * the blocks will be allocated separately, as the final size is
00007  * unknown. When compilation is done, the blocks will be copied into
00008  * the one big area.
00009  *
00010  * There are 5 different blocks of information for each program:
00011  * 1. The program itself. Consists of machine code instructions for a virtual
00012  *    stack machine. The size of the program must not be bigger than
00013  *    65535 bytes, as 16 bit pointers are used. Who would ever need a bigger
00014  *    program :-)
00015  * 2. Function names. All local functions that has been defined or called,
00016  *    with the address of the function in the program. Inherited functions
00017  *    will be found here too, with information of how far up the inherit
00018  *    chain that the function was defined.
00019  * 3. String table. All strings used in the program. They are all pointers
00020  *    into the shared string area. Thus, they are easily found and deallocated
00021  *    when the object is destructed.
00022  * 4. Table of variable names. They all point into the shared string table.
00023  * 5. Line number information. A table which tells at what address every
00024  *    line belongs to. The table has the same number of entries as the
00025  *    programs has source lines. This is used at errors, to find out the
00026  *    line number of the error.  This is usually swapped out to save space.
00027  *    First entry is the length of the table.
00028  * 6. List of inherited objects.
00029  */
00030 
00031 /*
00032  * When a new object inherits from another, all function definitions
00033  * are copied, and all variable definitions.
00034  * Flags below can't explicitly declared. Flags that can be declared,
00035  * are found with TYPE_ below.
00036  *
00037  * When an object is compiled with type testing NAME_STRICT_TYPES, all
00038  * types are saved of the arguments for that function during compilation.
00039  * If the #pragma save_types is specified, then the types are saved even
00040  * after compilation, to be used when the object is inherited.
00041  */

00043 /* The semantics of the flags are currently as follows:
00044 
00045    (1) A function is either a definition, a prototype or an undefined
00046        function
00047 
00048    (2) A definition is an address that executes code when
00049        it is jumped to and finishes with a return.
00050        The address is labeled by the following info:
00051        function name, type, flags,number of arguments, number of locals,
00052        argument types.
00053 
00054    (3) A prototype does not execute code, and is basically a label with
00055        the following info:
00056        argument types (optionally also argument names),function name,
00057        type, flags, number of arguments
00058 
00059    (4) An undefined function is an unknown function that has been called or
00060        a label that is as yet not known to be a prototype or definition
00061 
00062    Note that the above rules apply to comments in compiler.c as well.
00063    In particular, a known prototype is not undefined. When 'function'
00064    by itself is used, (1) is always meant.
00065 
00066    FUNC_INHERITED - The function entry that exists in this object actually
00067                     is a function in an object we inherited
00068    FUNC_UNDEFINED - see (4)
00069    FUNC_STRICT_TYPES - compiled with strict type testing
00070    FUNC_PROTOTYPE - see (3)
00071    FUNC_ALIAS     - This entry refers us to another entry, usually because
00072                     this function was overloaded by that function
00073 
00074    Sym
00075  */
