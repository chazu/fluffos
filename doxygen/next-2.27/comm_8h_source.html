00137  /*
00138   * This macro is for testing whether ip is still valid, since many
00139   * functions call LPC code, which could otherwise use
00140   * enable_commands(), set_this_player(), or destruct() to cause
00141   * all hell to break loose by changing or dangling command_giver
00142   * or command_giver->interactive.  It also saves us a few dereferences
00143   * since we know we can trust ip, and also increases code readability.
00144   *
00145   * Basically, this should be used as follows:
00146   *
00147   * (1) when using command_giver:
00148   *     set a variable named ip to command_giver->interactive at a point
00149   *     when you know it is valid.  Then, after a call that might have
00150   *     called LPC code, check IP_VALID(command_giver), or use
00151   *     VALIDATE_IP.
00152   * (2) some other object:
00153   *     set a variable named ip to ob->interactive, and save ob somewhere;
00154   *     or if you are just dealing with an ip as input, save ip->ob somewhere.
00155   *     After calling LPC code, check IP_VALID(ob), or use VALIDATE_IP.
00156   *
00157   * Yes, I know VALIDATE_IP uses a goto.  It's due to C's lack of proper
00158   * exception handling.  Only use it in subroutines that are set up
00159   * for it (i.e. define a failure label, and are set up to deal with
00160   * branching to it from arbitrary points).
00161   */
00162 #define IP_VALID(ip, ob) (ob && ip && ob->interactive == ip)
00163 #define VALIDATE_IP(ip, ob) if (!IP_VALID(ip, ob)) goto failure
