00003 /* this include file adds some nice debugging capabilities.  You can
00004    use -DDEBUG_MACRO in CFLAGS in the Makefile to let the debug code
00005    be compiled in.  If DEBUG_MACRO is not defined then the debug code
00006    is removed by the C preprocessor.  The global variable debug_level
00007    is defined in main.c.  The debug_level typically starts out at zero
00008    (0) and slowly grows larger over the life of the program.  The main
00009    advantage to this debug macro is that you can turn various sets of
00010    debug statements of and on by changing the value of debug_level.
00011    Say for example that debug_level is set to 7; then, all debug()
00012    statements with a first argument of 1, 2, or 4 would be printed. If
00013    debug_level is 6, then only debug()'s with 2 or 4 as the first arg
00014    would be printed.  Sample usage is as follows:
00015 
00016    debug(4,("module.c: simulate(): x = %d, y = %d\n",x,y));
00017 */
