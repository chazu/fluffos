00010 /* First, the polynomial itself and its table of feedback terms.  The  */
00011 /* polynomial is                                                       */
00012 /* X^32+X^26+X^23+X^22+X^16+X^12+X^11+X^10+X^8+X^7+X^5+X^4+X^2+X^1+X^0 */
00013 /* Note that we take it "backwards" and put the highest-order term in  */
00014 /* the lowest-order bit.  The X^32 term is "implied"; the LSB is the   */
00015 /* X^31 term, etc.  The X^0 term (usually shown as "+1") results in    */
00016 /* the MSB being 1.                                                    */
00017 
00018 /* Note that the usual hardware shift register implementation, which   */
00019 /* is what we're using (we're merely optimizing it by doing eight-bit  */
00020 /* chunks at a time) shifts bits into the lowest-order term.  In our   */
00021 /* implementation, that means shifting towards the right.  Why do we   */
00022 /* do it this way?  Because the calculated CRC must be transmitted in  */
00023 /* order from highest-order term to lowest-order term.  UARTs transmit */
00024 /* characters in order from LSB to MSB.  By storing the CRC this way,  */
00025 /* we hand it to the UART in the order low-byte to high-byte; the UART */
00026 /* sends each low-bit to hight-bit; and the result is transmission bit */
00027 /* by bit from highest- to lowest-order term without requiring any bit */
00028 /* shuffling on our part.  Reception works similarly.                  */
00029 
00030 /* The feedback terms table consists of 256, 32-bit entries.  Notes:   */
00031 /*                                                                     */
00032 /*     The table can be generated at runtime if desired; code to do so */
00033 /*     is shown later.  It might not be obvious, but the feedback      */
00034 /*     terms simply represent the results of eight shift/xor opera-    */
00035 /*     tions for all combinations of data and CRC register values.     */
00036 /*                                                                     */
00037 /*     The values must be right-shifted by eight bits by the "updcrc"  */
00038 /*     logic; the shift must be unsigned (bring in zeroes).  On some   */
00039 /*     hardware you could probably optimize the shift in assembler by  */
00040 /*     using byte-swap instructions.                                   */
00041
